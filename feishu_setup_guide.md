# 飞书多维表格 (Bitable) 配置与 CLI 使用指南

为了使虚拟招聘系统能够正常运行，您需要按照以下步骤在飞书平台上进行配置，并安装官方 CLI 工具。

## 第一步：安装官方 CLI 工具

本项目使用飞书官方提供的 `@larksuite/cli` 进行数据操作。

```bash
# 安装 CLI
npm install -g @larksuite/cli

# 验证安装
lark-cli --version
```

## 第二步：创建飞书自建应用
...
1.  登录 [飞书开放平台](https://open.feishu.cn/app?lang=zh-CN)。
2.  点击 **“创建自建应用”**，输入应用名称（如：虚拟招聘助手）和描述，点击创建。
3.  在左侧导航栏选择 **“凭证与基础信息”**，获取 `App ID` 和 `App Secret`。
4.  在左侧导航栏选择 **“权限管理”**，搜索并开启以下权限：
    *   `查看、评论及导出多维表格`
    *   `阅读和写入多维表格中的内容`
    *   `获取多维表格元数据`
5.  在左侧导航栏选择 **“应用发布”** -> **“版本管理与发布”**，创建一个版本并申请发布。由租户管理员审核通过后即可生效。

## 第二步：准备多维表格 (Base)

1.  在飞书云文档中新建一个 **“多维表格”**。
2.  获取 `Base ID`：在多维表格的 URL 中，`https://xxx.feishu.cn/base/` 之后的一串字符即为 `Base ID`。
3.  **授权应用**：点击多维表格右上角的 **“...”** -> **“更多”** -> **“添加文档应用”**，搜索并添加您刚刚创建的应用，确保应用有权操作此表格。

## 第三步：环境自动化初始化 (推荐)

**本项目已实现全自动环境搭建**。您无需在飞书多维表格中手动创建任何表或字段。

1.  确保您已完成“第二步：准备多维表格”中的 **授权应用** 操作（关键！）。
2.  运行项目入口脚本：
    ```powershell
    $env:PYTHONPATH="."; python src/main.py
    ```
3.  **系统行为**：
    - HR Agent 会首先扫描您的 Base。
    - 如果发现缺失必要的表（如 `岗位描述`、`简历池` 等），系统会自动通过 `lark-cli` 调用 API 按照预设的 Schema（包含所有正确字段类型）为您创建。
    - 初始化完成后，系统将直接进入简历生成与招聘流程。

## 第四步：配置项目环境变量

在项目根目录下创建 `.env` 文件（或修改 `.env.example`），填入您在上述步骤中获取的信息：

```env
# 飞书 API 配置
FEISHU_APP_ID=您的AppID
FEISHU_APP_SECRET=您的AppSecret
FEISHU_BASE_ID=您的BaseID

# 大模型配置
OPENAI_API_KEY=您的APIKey
OPENAI_API_BASE=您的API代理地址(可选)
MODEL_NAME=gpt-4o

# 招聘配置
NUM_CANDIDATES=3
JD_CONTENT="请输入您的岗位描述内容..."
```

## 注意事项
*   **表名必须完全一致**：系统初始化时会根据名称搜索 `简历池`、`面试官可用时间` 等。
*   **字段名称必须完全一致**：代码中使用了 Pydantic Alias 进行映射，字段名错误会导致数据写入失败。
*   **权限确认**：如果系统报错 `403` 或 `Permission Denied`，请检查“添加文档应用”步骤是否完成。
