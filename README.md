# 一、竞赛背景与目标

在企业实际运行中，业务的核心并不是单一工具，而是：

> **多个角色围绕数据协作，通过系统推动流程，并基于数据进行持续决策与优化。**

飞书多维表格（Base）作为业务系统构建平台，已经具备：

- 灵活的数据结构与关系建模能力
- 自动化流程与权限控制能力
- 可视化分析与协作能力

与此同时，大语言模型（LLM）与 AI Agent 技术的发展，使得：

- AI 可以承担“员工角色”，执行具体业务任务
- 系统可以从“被动记录”升级为“主动运行”

本次竞赛聚焦于这一前沿方向：

> **构建一个由 Agent 员工组成的虚拟组织，这些 Agent 作为“数字员工”，通过多维表格 OpenAPI 协同工作，完成系统搭建、业务运行与数据分析。**

# 二、竞赛核心任务

参赛团队需要完成一个**虚拟员工系统的构建与运行**，其中：

- 多个 AI Agent 扮演不同业务角色（员工）
- 所有业务通过飞书多维表格（Base）进行数据沉淀与状态管理
- Agent 通过飞书和多维表格 OpenAPI 操作多维表格，实现协同工作
- 系统能够持续运行并生成数据
- 最终基于数据完成分析、报告或决策输出

# 三、系统能力范围与要求

本课题围绕“组织运行系统”，要求参赛作品至少覆盖以下四个核心能力模块：

1. ## 虚拟员工建模

### 要求：

- 至少定义 **3 个及以上角色（Agent）**
- 每个 Agent 对应一个“员工角色”，例如：
    - 运营 / 编辑 / 审核 / 管理者
    - HR / 面试官 / 候选人
    - 用户 / 客服 / 风控

### 必须包含：

- 角色职责定义
- 输入 / 输出说明
- 角色间协作关系

1. ## 业务系统构建

Agent 角色基于飞书多维表格构建业务系统：

### 必须实现：

- 数据表设计（实体建模）
- 字段设计（结构化表达）
- 表间关系（关联记录）
- 状态字段（流程驱动）

### 技术要求：

- 必须让 Agent 角色通过 **多维表格 OpenAPI / SDK/CLI** 操作数据

1. ## 业务运行与协同

构建一个完整业务流程，使 Agent 员工能协同完成任务：

### 流程要求：

必须形成完整链路：

> 数据产生 → 状态更新 → Agent处理 → 决策 → 反馈 → 再流转

### 示例流程：

- 招聘：简历 → 筛选 → 面试 → 评估 → Offer
- 内容：选题 → 生产 → 审核 → 发布 → 反馈

### 技术实现：

- Agent 员工必须通过调用多维表格 API：
    - 读取数据
    - 更新状态
    - 写入结果

1. ## 数据分析与报告

系统运行后，需具备数据分析能力，并由 Agent 员工通过多维表格和飞书发送：

### 必须实现：

- 基于业务数据的自动分析（LLM 或逻辑计算）
- 输出至少一种结果形式：
    - 周报 / 日报
    - 数据洞察
    - 决策建议

1. ## 参赛约束与公平性

为了保证竞赛的公平性与技术的专注度，所有参赛团队必须在以下框架内角逐：

- **模型限制**：本次竞赛统一使用**国内模型**作为系统搭建与评测所依赖的大语言模型，不得使用其他大语言模型服务。为保证竞赛聚焦于模型的应用与编排能力，严禁对模型进行任何形式的微调，包括但不限于全量微调、LoRA、PEFT、RLHF 等。允许通过 检索增强生成（RAG）、提示词工程（Prompt Engineering）、工具调用（Tool-use）编排 等方式提升模型表现。
- **资源与依赖**：我们不限制参赛者使用的本地计算资源，必须在交付物中清晰声明所有使用的第三方大模型（API 或开源模型）、开源框架、依赖库和数据集。评委会综合评估你对生态资源的利用与整合能力。
- **复现与反作弊**：提交的作品必须能在评测环境中被完整、确定性地复现。所有系统搭建过程必须通过调用真实的 CLI/SDK 与 API 完成，严禁模拟（Mock）任何功能、伪造日志或虚构结果。评测端将通过行为回放与动态探测进行交叉验证，任何作弊行为都将导致一票否决。

1. ## 交付物说明

- **交付物要求**：
    - **源代码**：完整的、可独立运行的自动化构建脚本与相关代码，需包含清晰的注释。
    - **可运行包**：包含所有依赖和配置，可被评测端一键执行的环境包。
    - **技术文档**：详细阐述你的方案架构、AI Agent 设计思路、Prompt 策略、以及对多维表格特性的理解与应用。
    - **测试报告**：说明你如何对自己的方案进行测试，并展示测试结果。
    - **演示材料**：演示文稿或视频，以及可访问的多维表格产物。
